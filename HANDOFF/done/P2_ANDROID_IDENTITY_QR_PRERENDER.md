# TASK: Android IdentityScreen QR Code Pre-Rendering

## Agent Role
Agent 4: Android UI Performance (low-risk, single-screen)

## Context
SCMessenger Android app. File:
`android/app/src/main/java/com/scmessenger/android/ui/identity/IdentityScreen.kt`

User feedback (verbatim): "won't scroll after the QR loads (which is slow - pre render if possible)"

The QR code is generated by calling `viewModel.getQrCodeData()` from a `LaunchedEffect` on the Main thread (line 50-56 of IdentityScreen.kt). The QR code itself is then rendered later by `QRCodeDisplay` (line 196-200). The `getQrCodeData()` call goes through UniFFI → Rust core → generates the QR payload string. This blocks the UI for a visible interval (typically 200-2000ms depending on device).

## The Problem
- QR generation is a synchronous FFI call on the Main dispatcher.
- It runs AFTER the screen is already drawn, so the user sees an empty space where the QR will appear, then a sudden render.
- The user wants the QR to be ready by the time they see the screen.

## Acceptance Criteria
- [ ] QR data is generated off the Main thread (Dispatchers.Default or IO).
- [ ] QR is available immediately when the screen opens (no visible delay).
- [ ] ViewModel survives configuration changes (rotation, dark mode toggle) — use `StateFlow` not local `remember`.
- [ ] The screen does not flicker or jump when QR arrives.
- [ ] No regression: existing refresh button, error handling, copy-to-clipboard still work.
- [ ] `./gradlew :app:assembleDebug -x lint --quiet` succeeds.

## Implementation Plan

### Step 1: Move QR generation to ViewModel
In `IdentityViewModel.kt`:
```kotlin
private val _qrCodeData = MutableStateFlow<String?>(null)
val qrCodeData: StateFlow<String?> = _qrCodeData.asStateFlow()

init {
    // Pre-warm: generate QR as soon as identity is loaded
    viewModelScope.launch(Dispatchers.Default) {
        if (identityInfo.value?.initialized == true) {
            val data = getQrCodeData()  // suspend or wrap in withContext
            _qrCodeData.value = data
        }
    }
}

fun refreshQrCode() {
    viewModelScope.launch(Dispatchers.Default) {
        _qrCodeData.value = getQrCodeData()
    }
}
```

### Step 2: Observe in Composable
In `IdentityScreen.kt`, replace lines 49-56:
```kotlin
// OLD: local LaunchedEffect on Main thread
var qrCodeData by remember { mutableStateOf<String?>(null) }
LaunchedEffect(identityInfo) {
    if (identityInfo?.initialized == true) {
        qrCodeData = viewModel.getQrCodeData()
    } else {
        qrCodeData = null
    }
}
```
with:
```kotlin
// NEW: collect from ViewModel's StateFlow
val qrCodeData by viewModel.qrCodeData.collectAsState()
```

### Step 3: Show a placeholder while QR loads (optional polish)
While `qrCodeData == null`, show a fixed-size grey box (256×256 dp) so layout doesn't jump when the QR arrives.
