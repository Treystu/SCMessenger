# SCMessenger v0.2.0 Phase Execution Prompts

Status: Ready for use  
Date: 2026-03-02  
Reference Plan: `docs/MILESTONE_PLAN_V0.2.0_ALPHA.md`

---

## How to use this document

1. Run one phase at a time.
2. Copy the full prompt block for that phase into your coding model.
3. Keep model and reasoning settings exactly as specified for consistency.
4. Require acceptance checks to pass before moving to the next phase.
5. For WS1 and later, carry forward unresolved residual risks from `docs/V0.2.0_RESIDUAL_RISK_REGISTER.md` and prove they are not regressed before declaring phase success.
6. After WS5, run WS5.5 intermission gate before starting WS6/WS7.
7. Do not implement Single Active Device/Tight Pairing in WS0-WS12; it is tracked as `WS13` for `v0.2.1` in `docs/V0.2.1_SINGLE_ACTIVE_DEVICE_TIGHT_PAIR_PLAN.md`.
8. After Post-WS12 closure, use the WS13 kickoff prompt in this file to transition into v0.2.1 safely.

All prompts below align with repo philosophy:

1. LoC-based planning only (no time-based estimates).
2. Complete started functionality before adding net-new work.
3. No regressions, no behavior drift, and no feature creep outside phase scope.

### Mandatory carry-forward preflight (WS1+)

Before executing WS1 or later, prepend this to the phase prompt:

```text
Residual risk carry-forward is mandatory.
1. Read docs/V0.2.0_RESIDUAL_RISK_REGISTER.md.
2. List unresolved residual risk IDs from earlier phases.
3. Verify this phase does not worsen any unresolved risk.
4. If a risk is changed, update docs/V0.2.0_RESIDUAL_RISK_REGISTER.md with new evidence/status.
```
