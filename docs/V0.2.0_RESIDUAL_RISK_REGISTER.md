## R-WS1-01 - In-memory swarm retry loop is not persistent by itself

- Phase: WS1
- Status: Open (known architecture split)
- Impact: Runtime retry state in `swarm.rs` is process-memory only; durable eventual delivery relies on persistent outbox + replay flows.
- Evidence:
  - In-memory pending retry map and loop: [core/src/transport/swarm.rs:670](/Users/christymaxwell/Desktop/Luke_Stuff/GitHub/SCMessenger/core/src/transport/swarm.rs:670), [core/src/transport/swarm.rs:711](/Users/christymaxwell/Desktop/Luke_Stuff/GitHub/SCMessenger/core/src/transport/swarm.rs:711)
  - Persistent outbox enqueue at message preparation: [core/src/lib.rs:945](/Users/christymaxwell/Desktop/Luke_Stuff/GitHub/SCMessenger/core/src/lib.rs:945)
- Next action:
  - WS3 should preserve outbox durability assumptions while integrating relay custody.
  - WS12 should add restart/crash integration tests that explicitly prove this contract.
