# Plan: Ensure OpenRouter Free Tier Compliance for SCMessenger Swarm Orchestrator

## Overview
The current orchestrator configuration (in `.mimocode/mimocode.json`) uses a *free router* provider which automatically selects among the available free models. No custom routing logic or cost accounting is required for the free tier because all listed models in the free router have a cost of `0`. The orchestrator must, however, guarantee that:

1.  **No paid models are ever selected** -- any attempt to use a paid model must be blocked.
2.  **Request context is sufficiently scoped** -- requests must not exceed the per‑model context limits (e.g., 256 k context for some models). The free router enforces this internally.
3.  **Restarts are safe** -- the orchestrator should be idempotent with respect to task execution; running the orchestrator again should not duplicate work that has already been completed.

## Tasks

1. **Audit the orchestrator implementation** -- locate the file(s) that implement task distribution and model selection.
   *Expected path:* `scmessenger/src/orchestrator.rs` (if present) or similar.

2. **Verify provider configuration** -- ensure that the `provider` field in `mimocode.json` points to the correct OpenRouter endpoint and that the API key header is set.

3. **Confirm all agent `model` settings** -- every agent definition (build, plan, compose, etc.) should reference only a free‑tier model (including the `free-router` default). If any agent points to a paid model (e.g., `gpt-3.5-turbo`), update it.

4. **Enforce single‑session concurrency** -- ensure that the orchestrator invokes agents serially, not in parallel, to avoid exceeding the per‑model context or concurrent request limits.

5. **Add idempotency checks** -- before beginning a task, check whether the corresponding `.progress.md` or any marker file exists; skip the task if it has already been processed.

6. **Verification** -- run a dry‑run with `--dry-run` or equivalent flag (if available). Inspect logs to confirm that all requests are routed via the free router and that no error about paid models occurs.

- Updated `mimocode.json` if any changes were needed.
